## 1. 개요

Everything Claude Code(ECC)는 Claude Code를 위한 종합 플러그인으로, 소프트웨어 개발 전 과정을 AI 에이전트로 자동화합니다. Anthropic 해커톤 수상작이며, 단일 플러그인 설치만으로 47개의 전문 에이전트, 181개의 스킬, 79개의 커맨드를 즉시 사용할 수 있습니다.

### 지원 플랫폼

- **Claude Code** (primary) — claude.ai/code
- **Cursor IDE**
- **OpenCode**
- **Codex** (macOS/CLI)
- **Gemini**

### 지원 언어 에코시스템

TypeScript · Python · Go · Rust · Java · Kotlin · C++ · Swift · PHP · Perl · Dart/Flutter · C#

## 2. 설치

### 2.1 플러그인 설치 (권장)

Claude Code 대화창에서 다음 두 줄을 실행합니다:

```
/plugin marketplace add https://github.com/affaan-m/everything-claude-code
/plugin install ecc@ecc
```

설치 후 커맨드는 `/ecc:plan`, `/ecc:tdd` 등 `ecc:` 접두사로 사용됩니다.

### 2.2 Rules 수동 설치 (필수)

플러그인은 Rules 파일을 자동 설치하지 않습니다. 최상의 경험을 위해 수동으로 설치해야 합니다.

**macOS/Linux:**
```bash
git clone https://github.com/affaan-m/everything-claude-code.git
cd everything-claude-code
npm install
./install.sh --profile full
```

**Windows PowerShell:**
```powershell
git clone https://github.com/affaan-m/everything-claude-code.git
cd everything-claude-code
npm install
.\install.ps1 --profile full
```

**언어별 선택 설치:**
```bash
# 특정 언어만 선택
./install.sh typescript python golang
./install.sh swift kotlin java
./install.sh rust cpp perl
```

**대상 플랫폼 지정:**
```bash
./install.sh --profile full --target cursor
./install.sh --profile full --target gemini
./install.sh --profile full --target codex
```

### 2.3 설치 확인

```
/ecc:plan "테스트 기능 추가"
```

성공적으로 설치되었다면 계획 수립이 시작됩니다.

## 3. 핵심 개념

ECC는 6가지 핵심 구성 요소로 이루어져 있습니다:

| 구성 요소 | 설명 | 위치 |
|-----------|------|------|
| **Agent** | 특정 작업에 특화된 하위 에이전트. 자동으로 위임(delegate)됩니다. | `agents/` |
| **Skill** | 워크플로우 정의와 도메인 지식. 스킬을 호출하면 Claude가 해당 방식대로 작업합니다. | `skills/` |
| **Command** | 슬래시(`/`) 커맨드 진입점. 스킬이나 에이전트를 불러냅니다. | `commands/` |
| **Rule** | 항상 적용되는 코딩 가이드라인. CLAUDE.md에 자동 주입됩니다. | `rules/` |
| **Hook** | 이벤트 기반 자동화. 파일 편집, 도구 실행 전후에 트리거됩니다. | `hooks/` |
| **MCP Server** | 외부 서비스(GitHub, 검색, 브라우저 등) 연동 프로토콜 서버. | `mcp-configs/` |

### 동작 방식

```
사용자 요청
    ↓
Command(/plan) 실행
    ↓
Skill 로드 (도메인 지식)
    ↓
Agent 위임 (필요 시 병렬 실행)
    ↓
Rule 적용 (항상)
    ↓
Hook 트리거 (자동)
```

### 핵심 원칙

1. **Agent-First** — 도메인 작업은 전문 에이전트에게 위임
2. **Test-Driven** — 구현 전 테스트 작성 (80%+ 커버리지)
3. **Security-First** — 보안을 타협하지 않음
4. **Plan Before Execute** — 복잡한 기능은 먼저 계획 수립

### 7.1 새로운 기능 구현 (TDD 기반)

```
Step 1: 계획 수립
/ecc:plan "사용자 인증 기능 추가 — JWT, refresh token 포함"

Step 2: TDD 시작 (테스트 먼저)

Step 3: 코드 작성
# Claude가 테스트 → 구현 → 테스트 통과 순서로 진행

Step 4: 코드 리뷰
/code-review

Step 5: 검증
/verify
```

**언제 사용하나요?**
- 새로운 API 엔드포인트 추가
- 데이터베이스 스키마 변경
- 인증/권한 로직 구현

## 8. Hook 시스템

Hook은 Claude Code 이벤트에 자동으로 반응하는 자동화 트리거입니다.

### Hook 유형

| 유형 | 트리거 시점 |
|------|------------|
| `PreToolUse` | 도구 실행 직전 |
| `PostToolUse` | 도구 실행 직후 |
| `Stop` | Claude 응답 완료 시 |
| `SessionStart` | 세션 시작 시 |
| `SessionEnd` | 세션 종료 시 |

### Hook 프로필 설정

```bash
# 환경변수로 프로필 설정
export ECC_HOOK_PROFILE=standard  # minimal | standard | strict

# 특정 Hook 비활성화
export ECC_DISABLED_HOOKS="pre:bash:tmux-reminder,post:edit:typecheck"
```

| 프로필 | 설명 |
|--------|------|
| `minimal` | 핵심 Hook만 실행 (빠른 세션) |
| `standard` | 기본 Hook 세트 (권장) |
| `strict` | 모든 보안·검증 Hook 실행 |

### ECC에 포함된 주요 Hook

- **세션 지속성**: 세션 종료 시 자동으로 컨텍스트 저장
- **타입 체크**: 편집 후 자동 타입 검사
- **보안 가드**: 위험한 명령 실행 전 경고
- **학습 추출**: 세션 완료 시 패턴 자동 추출

## 9. MCP 서버

ECC에 번들된 MCP 서버 6개가 자동으로 설정됩니다:

| MCP 서버 | 용도 | 주요 기능 |
|----------|------|-----------|

## 11. 지속적 학습 시스템

ECC v2는 인스팅트(Instinct) 기반의 지속적 학습을 제공합니다.

### 인스팅트란?

개발 세션에서 추출된 재사용 가능한 패턴과 결정사항입니다. 신뢰도 점수가 축적되면 자동으로 스킬로 진화합니다.

### 인스팅트 생명주기

```
패턴 추출 (/learn)
    ↓
pending 상태 (30일 TTL)
    ↓
여러 세션에서 반복 확인
    ↓
신뢰도 임계값 도달
    ↓
confirmed 인스팅트로 승격
    ↓
스킬로 클러스터링 (/evolve)
```

### 팀 간 인스팅트 공유

```
# 인스팅트 내보내기
/instinct-export
# → instincts.json 파일 생성

# 다른 팀원이 가져오기
/instinct-import instincts.json

# 프로젝트 스킬로 승격
/promote
```

## 12. 보안 (AgentShield)

AgentShield는 ECC의 보안 스캐닝 시스템입니다.

### 스캔 방법

```bash
# 빠른 스캔
npx ecc-agentshield scan

# 자동 수정
npx ecc-agentshield scan --fix

# 특정 디렉토리
npx ecc-agentshield scan --dir ./src

# Opus 모델로 심층 분석
npx ecc-agentshield scan --opus --stream

# 다양한 출력 형식
npx ecc-agentshield scan --format json
npx ecc-agentshield scan --format markdown
npx ecc-agentshield scan --format html
```

### 탐지 카테고리

| 카테고리 | 탐지 항목 |
|----------|-----------|
| 시크릿 | API 키, 비밀번호, 토큰, 인증서 (14가지 패턴) |
| 권한 | 과도한 권한 설정 |
| Hook 주입 | 악성 Hook 코드 |
| MCP 위험 | 안전하지 않은 MCP 서버 설정 |
| 에이전트 설정 오류 | 잘못된 에이전트 구성 |

### 커밋 전 보안 체크리스트

- [ ] 하드코딩된 시크릿 없음
- [ ] 모든 사용자 입력 검증
- [ ] SQL 인젝션 방지 (파라미터화 쿼리)
- [ ] XSS 방지 (HTML 이스케이프)
- [ ] CSRF 보호 활성화
- [ ] 인증/권한 검증
- [ ] 모든 엔드포인트에 레이트 리밋
- [ ] 에러 메시지에 민감한 정보 없음

## 13. 설정 및 최적화

### 13.1 패키지 매니저 설정

ECC는 다음 순서로 패키지 매니저를 자동 감지합니다:

1. `CLAUDE_PACKAGE_MANAGER` 환경변수
2. `.claude/package-manager.json`
3. `package.json`의 `packageManager` 필드
4. 락 파일 탐지 (npm/yarn/pnpm/bun)
5. 전역 설정 `~/.claude/package-manager.json`
6. 첫 번째로 찾은 패키지 매니저

```json
// .claude/package-manager.json
{
  "packageManager": "bun"
}
```

### 13.2 토큰 최적화

```json
// ~/.claude/settings.json
{
  "env": {
    "MAX_THINKING_TOKENS": "10000",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50"
  }
}
```

**모델 선택 가이드:**
- **Sonnet** — 80%+ 작업에 사용, 약 60% 비용 절감
- **Opus** — 복잡한 아키텍처 결정, 보안 스캔, 계획 수립

```bash
# Hook 프로필 조정
export ECC_HOOK_PROFILE=minimal  # 빠른 개발 시

# 불필요한 MCP 서버 비활성화
export ECC_DISABLED_MCPS="playwright"
```

### 13.3 ECC 설정 파일

설치 프로필은 `~/.claude/.claude/ecc-tools.json`에 저장됩니다:

```json
{
  "profile": "full",
  "packages": [
    "runtime-core",
    "workflow-pack",
    "agentshield-pack",
    "research-pack",
    "team-config-sync",
    "enterprise-controls"
  ]
}
```

| 패키지 | 포함 내용 |
|--------|-----------|
| `runtime-core` | 기본 에이전트, 스킬, Hook |
| `workflow-pack` | 워크플로우 커맨드 및 스킬 |
| `agentshield-pack` | 보안 스캐닝 |

## 14. 문제 해결 (FAQ)

### Q: Rules가 적용되지 않습니다.

**원인:** 플러그인은 Rules 파일을 자동 배포하지 않습니다.
**해결:** 저장소를 클론하고 `install.sh`를 수동 실행하세요.

```bash
git clone https://github.com/affaan-m/everything-claude-code.git
cd everything-claude-code
npm install && ./install.sh --profile full
```

---

### Q: Hook이 동작하지 않습니다.

**원인:** `plugin.json`에 hooks 필드를 추가하면 안 됩니다. Claude Code v2.1+는 `hooks.json`을 자동 로드합니다.
**해결:** `ECC_DISABLED_HOOKS` 환경변수를 확인하고, `ECC_HOOK_PROFILE`을 `standard`로 설정하세요.

```bash
export ECC_HOOK_PROFILE=standard
```

---

### Q: 토큰 소진이 너무 빠릅니다.

**해결 방법:**
1. MCP 서버 수 줄이기: `export ECC_DISABLED_MCPS="playwright,context7"`
2. Opus 대신 Sonnet 사용
3. 무관한 작업 사이에 `/clear` 실행
4. Hook 프로필 축소: `export ECC_HOOK_PROFILE=minimal`

---

### Q: `/multi-*` 커맨드가 실패합니다.

**원인:** `ccg-workflow` 패키지가 설치되지 않았습니다.
**해결:**
```bash
npm install -g ccg-workflow
# 또는
bun add -g ccg-workflow
```

---

### Q: Windows에서 install.sh가 실행되지 않습니다.

**해결:** PowerShell 스크립트를 사용하세요.

```powershell
.\install.ps1 --profile full
```

---

### Q: 에이전트가 자동으로 위임되지 않습니다.

**원인:** ECC의 AGENTS.md가 CLAUDE.md에 주입되지 않았을 수 있습니다.
**해결:** Rules 수동 설치를 확인하고, 설치 후 새 세션을 시작하세요.

```bash
# 설치 재확인
./install.sh --profile full --force
```

---

### Q: `/ecc:plan`과 `/plan`의 차이는 무엇인가요?

- `/ecc:plan` — 플러그인 방식 설치 시 (네임스페이스 접두사)
- `/plan` — 수동 설치(install.sh) 시 (접두사 없음)

두 방법을 모두 설치한 경우 `/plan`을 사용하면 됩니다.
